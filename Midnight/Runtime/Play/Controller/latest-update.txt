Here's a summary of every change made across all files:

Phase 1 -- Non-blocking batched uploads:

VertexBuffer.h/.cpp -- Added fillStaging (map/memcpy/flush without GPU submit), recordCopy (records vkCmdCopyBuffer into an external command buffer), and cleanupStaging (destroys only the staging buffer). Made cleanup idempotent with null checks.

IndexBuffer.h/.cpp -- Same three new methods and idempotent cleanup.

VkTerrain.h -- Replaced the monolithic uploadTerrainChunkToGpu with two functions:

prepareChunkUpload -- creates GPU-local + staging buffers, fills staging via memcpy, sets up SSBOs and descriptor sets. No GPU copy submission.
recordChunkCopies -- records VBO/IBO staging-to-device copies into the caller's command buffer.
TerrainController.h -- Added TerrainUploadBatch struct (fence, command buffer, in-flight slot list), DeferredGpuCleanup struct, new method declarations, and budget constant kMaxChunksPerUploadBatch = 3.

TerrainController.cpp -- Constructor creates a reusable fence + command buffer. serviceCpuReadyChunks now calls two phases:

retireCompletedUploads -- polls the fence (vkGetFenceStatus, no blocking), marks completed slots Resident, frees staging buffers, resets the fence for reuse.
buildAndSubmitUploadBatch -- if no batch is active, prepares up to 3 CpuReady chunks, records all copy commands into one command buffer, records a single TRANSFER -> VERTEX_INPUT barrier, submits once with the fence. Marks slots Uploading.
Phase 2 -- Deferred GPU resource deletion:

TerrainController.cpp evictChunk -- Instead of calling cleanupOne (which uses ShaderStorageBuffer::cleanup with vkQueueWaitIdle), moves all GPU resource handles into a DeferredGpuCleanup entry tagged with the current frame index. If the chunk is mid-upload, it force-waits the fence first (rare edge case).

flushDeferredDeletes -- Called at the start of update() each frame, destroys entries that are at least kDeferFrames (3) frames old, guaranteeing the GPU has finished using them.

cleanup (shutdown) -- Waits for any in-flight batch, destroys the fence/command buffer, cleans up all live slots and deferred entries, then destroys descriptor pools.
